# Summary: Plan 2.2 — AdminEquipmentPanel Fixes

## What Was Done

Applied 9 targeted bug fixes to `src/pages/AdminEquipmentPanel.jsx` (826 lines). All fixes
were non-architectural single-line or small-block changes. No visual redesign; existing
Tailwind CSS styling preserved throughout.

The fixes address three categories of defect: (1) data corruption from non-deterministic
IDs, (2) silent failures in all three admin action handlers, and (3) null/undefined crashes
on missing GAS field values.

## Bugs Fixed (with bug IDs from RESEARCH.md)

| Bug ID | Description | Fix Applied |
|--------|-------------|-------------|
| BUG-ADMIN-01 | Random `BRAK-ID-${Math.random()}` on every fetch corrupts issuing, return, and issuedItemIds set | Replaced with deterministic `SSUEW-BRAK-${item.NAZWA_SPRZĘTU \|\| 'X'}` |
| BUG-ADMIN-02 | `resolveFirstAidReport` filtered by `r.ID` only; empty catch swallowed all errors | Now parses `result.success`; filter uses `r.ID \|\| r.id`; catch shows error alert |
| BUG-ADMIN-03 | Four-name fallback chain for `apteczkiBraki` silently returns `[]` when GAS key differs | Added `console.log('[CRW] fetchAllData keys:', Object.keys(data))` before the chain |
| BUG-ADMIN-04 | Hardcoded `"01/SSUEW/03/2026"` fallback caused duplicate document numbers | `setDocNumber('BŁĄD — odśwież')` on failure; "Generuj PDF" button disabled; error note shown |
| BUG-ADMIN-05 | `confirmAndSendInvite` catch block told user "success" even on network failure | Catch now shows: "Błąd połączenia — wniosek NIE został zatwierdzony. Spróbuj ponownie." |
| BUG-ADMIN-06 | `handleRejectReservation` had empty `catch (err) {}` — rejections silently lost | Catch now shows: "Błąd połączenia — odrzucenie NIE zostało zapisane. Spróbuj ponownie." |
| BUG-ADMIN-08 | Return table: `String(undefined \|\| undefined)` rendered literal string "undefined" | Added `\|\| ''` fallback + `.filter(Boolean)` to skip empty entries |
| BUG-ADMIN-09 | `initiateApproval` called `.match()` directly on `rez.Kontakt` — throws if null | Changed to `(rez.Kontakt \|\| '').match(...)` |
| ADMIN-MISSING-05 | Apteczki badge was a pulsing dot that disappeared inside the apteczki tab | Replaced with always-visible numeric count badge (`firstAidReports.length`) |

  prevents empty table rows when the equipment string has trailing commas.
- The apteczki badge no longer uses `animate-ping` at all — the count badge communicates
  urgency more clearly and works equally well inside and outside the tab.
- BUG-ADMIN-07 (processReturn sending wrong `nrPorozumienia`) was out of scope for this
  plan — it requires a GAS field-name audit. Tracked in deferred items.

  removed once the canonical GAS key for `apteczkiBraki` is confirmed in production.
