# Summary: Plan 1.4 — Request-Access Sender Fix

## What Was Done

Fixed both hardcoded `onboarding@resend.dev` sender addresses in `api/request-access.js` by importing the shared `FROM` constant from `api/_email.js`. Applied `esc()` to all six user-supplied request body fields in the admin notification email HTML, and to `name` in the applicant confirmation email HTML. The import line, both send-call `from` fields, and all six `esc()` wrappings were the only changes — validation logic, Firestore write, subject lines (except name in admin subject now uses `esc(name)`), HTML structure, and table layout are all unchanged.

  - Applicant confirmation email HTML: `${name}` → `${esc(name)}`
