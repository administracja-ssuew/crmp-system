# Summary: Plan 1.1 — Shared Email Helper

## What Was Done

Created `api/_email.js` as a pure string-utility helper module for all three Resend transactional email handlers. The file uses the Vercel underscore-prefix convention (`_email.js`) to prevent Vercel from exposing it as a serverless endpoint while allowing sibling handlers to import it via relative ESM imports.

The helper centralises four concerns that were previously duplicated or missing across `approve-request.js`, `reject-request.js`, and `request-access.js`:

1. **`FROM` constant** — reads `process.env.RESEND_FROM_EMAIL` with a `noreply@example.com` fallback that causes Resend to reject loudly if the env var is unset, rather than silently falling back to the sandbox address.
2. **`esc(str)`** — HTML-escapes user-supplied strings (`&`, `<`, `>`, `"`) to prevent HTML injection when `name`/`email` from `req.body` is interpolated into email bodies.
3. **`emailShell(title, bodyHtml)`** — returns a complete HTML document with table-based layout (Outlook-compatible), inline styles only (Gmail strips `<style>` blocks), and the SSUEW brand header/footer.
4. **`ctaButton(label, href)`** — returns a styled `<a>` tag with the brand color `#4f46e5`.

No SDK imports, no Firebase references, no `export default` — the file is a pure helper.

## Notes

**ctaButton signature deviation from RESEARCH.md:** The RESEARCH.md showed `ctaButton(href, label)` (href first), but the PLAN.md specified `ctaButton(label, href)` (label first, matching natural reading order at call sites). The PLAN.md content was used as authoritative — it is the implementation spec.

**`emailShell` title escaping:** The `<title>` tag in the HTML head runs `${esc(title)}`, meaning the email subject passed as `title` is HTML-escaped before insertion. The `bodyHtml` argument is injected raw — callers must apply `esc()` to any user input within `bodyHtml` before passing it to `emailShell()`.

**No stubs:** All four exports are fully implemented and ready to import. No placeholder values, no TODOs in the created file.
